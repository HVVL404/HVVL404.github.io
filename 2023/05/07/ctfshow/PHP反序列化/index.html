web 263

这题考查Session反序列化，还是值得一做的，首先讲一下思路，一开始是源码泄露 www.zip，然后代码审计，发现在 index.php 里有 session 值和 cookie 值，代码如下：

可以发现 cookie 值是可控的，也就意味着 Session 可控，而且需要 base64 编码，再看 /inc/inc.php ，发现了最重要的

这个就表示该 php 代码的序列化的引擎是 php，具体的知识点请移步我写的 Web漏洞-php反序列化，这里就不再赘述。从此就可以看出是 Session 反序列化了，那么我们知道 php 的利用方式是（键值+’|’+序列化的值），在 inc.php 里就有给出了 User 类，且 username 和 password 可控，利用点就是 file_put_contents。

现在思路是在 cookie 的 limit 给出你的 payload，然后携带这个 cookie 去访问index.php，他就会去修改 Session 的值，当然了，check.php 会去校验你写的是否合规，然后我们最后去访问 /inc/inc.php，目的是为了触发会话 Session 反序列化，最后就是调用到 file_get_contents 写入 shell 即可，下面给出 poc。
